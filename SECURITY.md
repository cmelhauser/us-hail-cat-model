# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.1.x   | ✅ Active |
| 2.0.x   | ⚠️  Critical fixes only |
| 1.x     | ❌ End of life |

## Reporting a Vulnerability

This is a scientific modeling repository. The primary security surface is:

- **Third-party data downloads** (S3, CDS API, NCAR RDA) — scripts do not
  validate checksums by default; a compromised upstream could alter model inputs.
- **Pickle artifacts** (`gridrad_cqm_model.pkl`, `hail_filter_model.pkl`) —
  pickle files can execute arbitrary code on load. Only load `.pkl` files
  generated by your own trusted pipeline run.
- **External package dependencies** — the model relies on scientific Python
  packages; keep dependencies updated.

To report a security issue, please email:

**christopher.melhauser@gmail.com**

Include:

1. A description of the vulnerability.
2. Steps to reproduce.
3. Potential impact.
4. (Optional) a suggested fix.

You will receive a response within 5 business days. Please do not open a public
GitHub issue for security vulnerabilities.
