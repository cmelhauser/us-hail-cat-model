# CONUS Hail Catastrophe Model

## What This Produces

This model builds a full **probabilistic hail hazard layer** for the contiguous United States (CONUS):

- **Return period maps** — Hail size (inches) at 10, 25, 50, 100, 200, 250, and 500-year return periods for every 0.05° grid cell (~5.5 km) across CONUS.
- **Annual occurrence probability rasters** — Probability that hail exceeds 0.25", 0.50", 1.00", 1.50", 2.00", 3.00", 4.00", or 5.00" at each cell in a given year.
- **Event catalog** — Discrete historical storm events (1998–present) identified using a synoptic-system grouping rule: consecutive hail days are merged if footprints overlap within 83 km and duration ≤ 5 days. Each event records dates, footprint, peak hail, and centroid lat/lon.
- **Daily climatology** — 366 daily climatology rasters capturing seasonal variation.
- **50,000-year stochastic catalog** — Generated by event-resampling from the historical catalog with calibrated log-normal intensity perturbation and spatial translation. Preserves real spatial footprint geometry. Produces occurrence and aggregate Probable Exceedance Tables (PETs).
- **Vulnerability curves** — Lognormal mean damage ratio (MDR) by construction class and roof type (placeholder — requires claims data for full calibration).
- **MESH validation report** — Cross-validation of radar-derived hail sizes against SPC ground reports, documenting bias correction effectiveness and population-independence of the hazard layer.

The model uses **no commercial hazard data** — only publicly available NOAA MRMS/MYRORSS radar data, NOAA SPC storm reports (for validation), and US Census Bureau population estimates (for validation only).

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_ACCOUNT/us-hail-cat-model.git
cd us-hail-cat-model
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

Python 3.9+ required. The pipeline checks all dependencies at startup and fails fast with a clear message if any are missing.

### 3. Run the pipeline

```bash
python run_pipeline.py
```

All 15 stages run in order (~10–12 hours total, dominated by download and raster construction). Progress and timing printed per stage. If a stage fails, the pipeline stops and tells you how to resume:

```bash
python run_pipeline.py --from 4       # resume from stage 4
python run_pipeline.py --only 9       # re-run a single stage
python run_pipeline.py --skip 14,15   # skip stages
python run_pipeline.py --dry-run      # preview without running
python run_pipeline.py --validate     # check outputs only, no computation
```

> **Note:** Stages 1, 2, 4, and 13 skip existing files and are safe to re-run after interruption.

## Pipeline

| Step | Script | What it does | Approx runtime |
|---|---|---|---|
| 1 | `01_download_myrorss.py` | Download MYRORSS MESH data (1998–2011) from AWS S3 | ~2–6 hrs |
| 2 | `02_download_mrms_mesh.py` | Download operational MRMS MESH data (2012–present) | ~2–6 hrs |
| 3 | `03_download_spc.py` | Download NOAA SPC daily hail CSVs (validation/calibration) | ~5 min |
| 4 | `04_build_mesh_rasters.py` | Aggregate native MESH to 0.05° daily GeoTIFFs (block-max) | ~2 hrs |
| 5 | `05_apply_mesh_bias_correction.py` | MESH75 recalibration + environmental filtering | ~1 hr |
| 6 | `06_validate_mesh_vs_spc.py` | Cross-validate corrected MESH against SPC ground reports | ~15 min |
| 7 | `07_build_hail_climo.py` | Build 366-file daily climatology at 0.05° | ~10 min |
| 8 | `08_build_event_catalog.py` | Identify events (synoptic grouping), build event catalog | ~15 min |
| 9 | `09_fit_cdf_regional.py` | Fit lognormal+GPD CDFs with regional ξ pooling + MRL diagnostics | ~30 min |
| 10 | `10_build_smooth_cdf.py` | Spatially-pooled CDF rebuild (150 km kernel) | ~30 min |
| 11 | `11_build_occurrence_probs.py` | Build annual occurrence probability rasters (8 thresholds) | ~10 min |
| 12 | `12_apply_conus_mask.py` | Apply CONUS mask + topographic correction | ~10 min |
| 13 | `13_generate_stochastic_catalog.py` | 50,000-yr event-resampling catalog with calibrated perturbation | ~3 hrs |
| 14 | `14_build_vulnerability.py` | Build MDR vulnerability curves by construction class | ~5 min |
| 15 | `15_render_figures.py` | All figures: historical, stochastic, validation, EP curves | ~45 min |

Each stage validates its outputs on completion and exits with code 1 if any files are corrupt or missing, stopping the pipeline chain.

## Known Limitations

1. **MESH overestimation bias:** MESH75 recalibration reduces but does not eliminate the radar tendency to overestimate hail size. Corrected MESH values should be interpreted as calibrated estimates, not direct measurements.

2. **West Coast radar coverage:** GridRad/MRMS coverage is sparse west of the Cascades and Sierra Nevada. West Coast cells appear as nodata — this is physically correct (negligible severe hail) and consistent with the Murillo et al. (2021) climatology.

3. **28-year record for CDF fitting:** Extending the record from 22 to 28 years improves GPD tail stability (~30% more data) but return periods beyond ~100 years still carry significant extrapolation uncertainty.

4. **Event-resampling template library:** Cannot generate truly novel footprint geometries not present in the 28-year record. Spatial translation partially addresses this. A longer record or learned spatial perturbation model would further improve this.

5. **Vulnerability curves are placeholder:** MDR parameters are from published literature, not calibrated to actual claims data. Production use requires proprietary claims-data calibration.

6. **No exposure layer:** The model produces hazard only. Converting to insured loss (EP curve, AAL) requires a TIV exposure database by location and construction class.

7. **Stationarity assumption:** The model assumes the hail climate is stationary over the record period. Trend diagnostics are included but trends are not incorporated into the CDF fitting.
